1. My team will use the branch formatting
    a. Development
    b. feature/
    c. hotfix/
    d. release
2. Start your commit message with what is happening ex. "Added", "Changed", "Deleted" then adding a description of what was changed and why
3. Pull requests should be made to make sure there are no conflicts and if there are try to resolve them before the conflicts become too much of an issue
4. We will use first do features into development then into release and if we need to do hotfixes then it will be hotfix/ to development to release 
5. Common git commands are
    a. git add
    b. git commit
    c. git pull
    d. git status
    e. git branch
    f. git checkout
